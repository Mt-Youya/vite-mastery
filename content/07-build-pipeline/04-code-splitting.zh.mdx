## 动态 import 自动分割

任何动态 `import()` 都会自动触发代码分割:

```ts
// 路由懒加载(以 React Router 为例)
const Home = lazy(() => import("./pages/Home"))
const About = lazy(() => import("./pages/About"))
const Admin = lazy(() => import("./pages/Admin"))
```

构建后产物:

```text
dist/
  assets/
    index-HASH.js         ← 入口 chunk(包含路由配置)
    Home-HASH.js          ← 按需加载的页面 chunk
    About-HASH.js
    Admin-HASH.js
    vendor-HASH.js        ← 共享依赖(react 等)
```

用户访问首页时只下载 `index.js` 和 `Home.js`,其他页面按需下载。

## `manualChunks`:手动控制分割

### 对象格式(简单)

```ts filename="vite.config.ts"
export default defineConfig({
  build: {
    rolldownOptions: {
      output: {
        manualChunks: {
          // 指定 chunk 名 → 包含的模块(包名或文件路径)
          "react-vendor": ["react", "react-dom", "react-router-dom"],
          "ui-vendor": ["@radix-ui/react-dialog", "@base-ui/react"],
          utils: ["lodash-es", "date-fns", "axios"],
        },
      },
    },
  },
})
```

### 函数格式(灵活)

```ts
manualChunks(id: string) {
  // id 是模块的文件路径

  if (id.includes("node_modules")) {
    // 把所有 node_modules 按包名分割
    const match = id.match(/node_modules\/([^/]+)\//)
    const packageName = match?.[1]

    if (!packageName) return undefined

    // react 生态放一起
    if (packageName.startsWith("react") || packageName === "scheduler") {
      return "react-vendor"
    }

    // Radix UI 组件库
    if (packageName.startsWith("@radix-ui")) {
      return "radix-vendor"
    }

    // 工具库
    if (["lodash-es", "date-fns", "dayjs"].includes(packageName)) {
      return "utils"
    }

    // 其他小包合并进 vendor
    return "vendor"
  }

  // 按路由分割页面代码
  if (id.includes("/pages/")) {
    const pageName = id.match(/\/pages\/([^/]+)\//)?.[1]
    if (pageName) return `page-${pageName}`
  }
},
```

## chunk 命名策略

```ts filename="vite.config.ts"
export default defineConfig({
  build: {
    rolldownOptions: {
      output: {
        // 动态 chunk(懒加载的)的文件名
        chunkFileNames: (chunkInfo) => {
          const { name } = chunkInfo
          if (name.startsWith("page-")) {
            return "pages/[name]-[hash].js"
          }
          return "assets/[name]-[hash].js"
        },

        // 入口 chunk 的文件名
        entryFileNames: "assets/[name]-[hash].js",

        // 静态资源(图片、字体等)的文件名
        assetFileNames: "assets/[name]-[hash][extname]",
      },
    },
  },
})
```

## 分析 bundle

使用 `rollup-plugin-visualizer` 分析产物:

```bash
pnpm add -D rollup-plugin-visualizer
```

```ts filename="vite.config.ts"
import { defineConfig } from "vite"
import { visualizer } from "rollup-plugin-visualizer"

export default defineConfig({
  plugins: [
    visualizer({
      open: true, // 构建后自动打开
      gzipSize: true, // 显示 gzip 后大小
      brotliSize: true, // 显示 brotli 后大小
      filename: "stats.html",
    }),
  ],
})
```

运行 `pnpm build`,自动打开可视化图表——能直观看到哪个包占用了最多体积。

## 预加载策略

Vite 自动生成预加载提示,减少懒加载时的延迟:

```html
<!-- Vite 在 HTML 里自动注入 modulepreload -->
<link rel="modulepreload" href="/assets/react-vendor-HASH.js" />
<link rel="modulepreload" href="/assets/index-HASH.js" />
```

可以禁用:

```ts
export default defineConfig({
  build: {
    modulePreload: false, // 禁用预加载
    // 或
    modulePreload: {
      polyfill: false, // 不注入 modulepreload polyfill
    },
  },
})
```
