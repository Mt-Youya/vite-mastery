## 什么是 Module Federation

Module Federation(模块联邦)允许多个独立的 JavaScript 应用在**运行时**互相共享模块——不是在构建时打包在一起,而是在浏览器里动态加载。

```text
应用 A(Host)        应用 B(Remote)
─────────────────   ─────────────────
部署在 app-a.com    部署在 app-b.com

A 在运行时动态 import B 导出的组件:
await import("app-b/Button")  // 从远程加载!
```

主要用途:

- **微前端**:多个团队各自维护一个子应用,汇聚到一个 shell 应用
- **共享组件库**:多个应用共享同一套 UI 组件,更新时无需重新部署消费方

## 在 Vite 8 中使用 Module Federation

Vite 8 通过 `@module-federation/vite` 插件支持 Module Federation:

```bash
pnpm add -D @module-federation/vite
```

### Remote 应用(暴露模块)

```ts filename="app-b/vite.config.ts"
import { defineConfig } from "vite"
import { federation } from "@module-federation/vite"

export default defineConfig({
  plugins: [
    federation({
      name: "app-b", // remote 应用名称

      // 声明这个应用暴露哪些模块
      exposes: {
        "./Button": "./src/components/Button",
        "./utils": "./src/utils/index",
      },

      // 声明哪些包是共享的(Host 和 Remote 共用同一份)
      shared: {
        react: { singleton: true },
        "react-dom": { singleton: true },
      },
    }),
  ],
})
```

### Host 应用(消费远程模块)

```ts filename="app-a/vite.config.ts"
import { defineConfig } from "vite"
import { federation } from "@module-federation/vite"

export default defineConfig({
  plugins: [
    federation({
      name: "app-a",

      // 声明依赖哪些 remote 应用
      remotes: {
        "app-b": "app-b@https://app-b.example.com/assets/remoteEntry.js",
      },

      shared: {
        react: { singleton: true },
        "react-dom": { singleton: true },
      },
    }),
  ],
})
```

### 在 Host 应用中使用远程模块

```tsx filename="app-a/src/App.tsx"
import { lazy, Suspense } from "react"

// 异步加载来自 Remote 的组件
const RemoteButton = lazy(() => import("app-b/Button"))

export function App() {
  return (
    <Suspense fallback={<div>加载中...</div>}>
      <RemoteButton>来自 app-b 的按钮</RemoteButton>
    </Suspense>
  )
}
```

## 共享依赖(Shared)

`shared` 配置是 Module Federation 的关键——防止 React 被打包两份:

```ts
shared: {
  react: {
    singleton: true,   // 只允许一个实例(防止多版本冲突)
    requiredVersion: "^19.0.0",  // 版本要求
    eager: true,       // 提前加载(不懒加载)
  },
}
```

如果 Host 和 Remote 都用了 React,`singleton: true` 确保浏览器里只有一个 React 实例——这对 hooks 和 context 至关重要。

## 与 Webpack Module Federation 的关系

Vite 的 `@module-federation/vite` 与 Webpack 5 的 Module Federation 格式**兼容**:

- Webpack 5 应用可以消费 Vite 构建的 Remote
- Vite 应用可以消费 Webpack 5 构建的 Remote
- 共用同一套配置概念(`name` / `exposes` / `remotes` / `shared`)

## 适用场景与局限

**适合**:

- 大型企业应用,多个团队独立开发子应用
- 希望独立部署、独立发布的模块
- 已有 Webpack Module Federation 架构需要迁移到 Vite

**不适合**:

- 小型应用(引入了复杂性但收益有限)
- 对性能要求极高的场景(运行时网络请求有开销)
- 需要 SSR 的场景(Module Federation + SSR 集成较复杂)
