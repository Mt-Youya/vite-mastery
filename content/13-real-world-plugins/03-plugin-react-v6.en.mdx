## Plugin Structure Overview

```ts
// Core structure of @vitejs/plugin-react v6 (simplified)
export default function pluginReact(options?: Options): Plugin[] {
  return [
    // Plugin 1: Inject Fast Refresh runtime into the entry HTML
    {
      name: "vite:react-refresh-preamble",
      enforce: "pre",
      transformIndexHtml: injectRefreshRuntime,
    },

    // Plugin 2: Transform JSX + inject Fast Refresh HMR code
    {
      name: "vite:react-transform",
      enforce: "pre",
      transform: transformReact,
    },
  ]
}
```

## Core Transform Flow

```ts
async function transformReact(code: string, id: string) {
  // 1. Filter: only process React files
  if (!id.match(/\.(jsx?|tsx?)$/)) return null
  if (id.includes("node_modules")) return null

  // 2. Check whether the file contains JSX (fast path: skip if no JSX)
  const hasJSX = code.includes("<") || code.includes("React.createElement")
  if (!hasJSX && !code.includes("import React")) return null

  // 3. Call OXC to perform JSX transform + TypeScript type stripping
  const transformed = await oxcTransform(code, {
    // JSX transform config (corresponding to the React 17+ new JSX transform)
    jsx: "react-jsx",
    jsxImportSource: "react",
    // TypeScript support
    typescript: true,
  })

  // 4. Inject React Fast Refresh code
  if (shouldInjectRefresh(transformed.code)) {
    return injectFastRefresh(transformed)
  }

  return transformed
}
```

## Fast Refresh Injection

```ts
function injectFastRefresh(result: TransformResult): TransformResult {
  const { code, map } = result

  // Detect which React components are in the file
  const components = detectComponents(code)

  if (components.length === 0) return result

  // Inject Fast Refresh code
  const refreshCode = `
import RefreshRuntime from "/@react-refresh"

// Register each component
${components.map((c) => `RefreshRuntime.register(${c.name}, ${JSON.stringify(c.name)})`).join("\n")}

// Accept HMR updates
if (import.meta.hot) {
  import.meta.hot.accept()
  // Notify React to perform Fast Refresh
  if (!window.$RefreshReg$) {
    import.meta.hot.invalidate()
  } else {
    RefreshRuntime.performReactRefresh()
  }
}
`

  return {
    code: code + "\n\n" + refreshCode,
    map: mergeSourceMaps(map, refreshCode),
  }
}
```

## The `/@react-refresh` Virtual Module

This is the host for the Fast Refresh runtime:

```ts
// Inject the preamble in transformIndexHtml
function injectRefreshRuntime(html: string): string {
  return html.replace(
    "<head>",
    `<head>
<script type="module">
import RefreshRuntime from "/@react-refresh"
RefreshRuntime.injectIntoGlobalHook(window)
window.$RefreshReg$ = () => {}
window.$RefreshSig$ = () => (type) => type
window.__vite_plugin_react_preamble_installed__ = true
</script>`,
  )
}

// Contents of the /@react-refresh virtual module
// In practice it is a thin wrapper around react-refresh/runtime
resolveId(id) {
  if (id === "/@react-refresh") return "\0@react-refresh"
  return null
},
load(id) {
  if (id !== "\0@react-refresh") return null
  return `
export * from "react-refresh/runtime"
import * as runtime from "react-refresh/runtime"
export default runtime
`
},
```

## The Key to Backward Compatibility

v6 maintains backward compatibility through these design choices:

1. **Config API unchanged**: The options signature of `react({ ... })` is essentially the same as v5
2. **Behavior unchanged**: The injected Fast Refresh code format is identical
3. **Babel fallback**: If you use `babel: { plugins: [...] }`, Babel is automatically used for processing
