## 坑一:workspace 包没有被预构建

**问题**:本地 `packages/ui` 是 TypeScript 源码(main 字段指向 `src/index.ts`),Vite 没有预构建它,每次 transform 都重新编译。

**根因**:Vite 的预构建只处理 `node_modules` 里的包。workspace 包通过软链接存在于 `node_modules`,但 Vite 知道它们是本地包,不预构建。

**解法一**:在 Next.js 里用 `transpilePackages`

```ts filename="apps/web/next.config.ts"
export default {
  transpilePackages: ["@vite-mastery/ui", "@vite-mastery/mdx-components"],
}
```

**解法二**:在 Vite 配置里用 `optimizeDeps.include`

```ts filename="vite.config.ts"
export default defineConfig({
  optimizeDeps: {
    include: ["@vite-mastery/ui > react"], // 包含间接依赖
  },
})
```

**解法三**:用 `resolve.alias` 直接指向 src/

```ts filename="vite.config.ts"
import { resolve } from "node:path"

export default defineConfig({
  resolve: {
    alias: {
      "@vite-mastery/ui": resolve("../../packages/ui/src"),
    },
  },
})
```

## 坑二:符号链接解析问题

**问题**:pnpm 使用符号链接管理 workspace 包,但有时候 Vite 的模块解析不能正确跟随符号链接。

**表现**:同一个包被解析为两个不同的模块(导致 `instanceof` 失败、React context 不共享等)。

**根因**:符号链接路径和真实路径被当作不同模块缓存。

**解法**:在 `vite.config.ts` 里启用符号链接解析:

```ts filename="vite.config.ts"
export default defineConfig({
  resolve: {
    // 把符号链接解析为真实路径
    preserveSymlinks: false, // 默认值,通常不需要改
  },
  server: {
    fs: {
      // 允许访问 monorepo 里所有包的文件
      allow: ["../.."],
    },
  },
})
```

## 坑三:CSS 变量跨包不生效

**问题**:`packages/tailwind-config/theme.css` 定义了 CSS 变量,但在 `packages/ui` 里使用时不生效。

**根因**:CSS 变量需要在 DOM 里定义,但包本身不引入 CSS 文件——最终由消费方 app 引入。

**解法**:在 `apps/web/app/globals.css` 里显式 import:

```css filename="apps/web/app/globals.css"
@import "@vite-mastery/tailwind-config/theme.css";
/* ↑ 确保 CSS 变量在应用级别注入 */
```

## 坑四:HMR 跨包不更新

**问题**:修改 `packages/ui/src/Button.tsx`,`apps/web` 里的页面没有 HMR 更新。

**根因**:Vite 只监听 `apps/web/` 目录下的文件变化,不监听 `packages/` 里的。

**解法**:配置 `server.watch`:

```ts filename="vite.config.ts"
export default defineConfig({
  server: {
    watch: {
      // 监听 packages/ 目录
      // (pnpm workspace 下软链接到 node_modules/.pnpm)
      followSymlinks: true,
    },
  },
})
```

或者在 `packages/ui` 里用 `vite build --watch` 保持构建产物更新,`apps/web` 消费构建产物。

## 坑五:TypeScript 找不到类型

**问题**:workspace 包有 `"main": "src/index.ts"` 但没有 `"types"` 字段,TypeScript 找不到类型。

**解法**:在 `package.json` 里正确声明 types 路径:

```json filename="packages/ui/package.json"
{
  "exports": {
    ".": {
      "import": "./src/index.ts",
      "types": "./src/index.ts" // 直接指向 TS 源码
    }
  }
}
```

或者在根 `tsconfig.json` 里配置路径映射:

```json filename="tsconfig.json"
{
  "compilerOptions": {
    "paths": {
      "@vite-mastery/ui": ["./packages/ui/src/index.ts"]
    }
  }
}
```
