## vite-plugin-inspect:插件级调试

`vite-plugin-inspect` 是 Vite 生态里最实用的调试工具:

```bash
pnpm add -D vite-plugin-inspect
```

```ts filename="vite.config.ts"
import { defineConfig } from "vite"
import Inspect from "vite-plugin-inspect"

export default defineConfig({
  plugins: [
    Inspect(),
    // ... 其他插件
  ],
})
```

启动 dev server 后访问 `http://localhost:5173/__inspect`:

### 能看到什么

1. **模块列表**:所有被加载的模块及其 transform 时间
2. **Transform 链路**:每个模块经过哪些插件处理、各耗时多少
3. **代码 Diff**:每个插件处理前后的代码对比
4. **插件图**:插件调用关系

### 典型使用场景

**场景一:某个文件加载特别慢**

在模块列表里按时间排序,找出耗时最长的模块。点进去看是哪个插件导致的。

**场景二:某个插件没有生效**

在模块的 transform 链路里找你的插件。如果不在列表里:

- 检查 `apply` 配置是否正确
- 检查文件过滤条件

**场景三:代码被意外修改**

用 Diff 模式查看每个插件的修改范围,找出修改了你不期望内容的插件。

## Chrome DevTools Performance

对于更底层的性能分析,直接用 Chrome DevTools:

```bash
# 开启 V8 profiler
node --prof $(which vite) dev
node --prof-process isolate-*.log > processed.txt
```

或者用 Clinic.js:

```bash
pnpm add -g clinic
clinic doctor -- node ./node_modules/.bin/vite dev
```

## 调试 transform 耗时

在自己的插件里加计时:

```ts
import type { Plugin } from "vite"

export function timedPlugin(wrappedPlugin: Plugin): Plugin {
  const original = wrappedPlugin.transform

  return {
    ...wrappedPlugin,
    transform(code, id, options) {
      const start = performance.now()
      const result = original?.call(this, code, id, options)
      const elapsed = performance.now() - start

      if (elapsed > 10) {
        console.log(`[${wrappedPlugin.name}] transform ${id}: ${elapsed.toFixed(1)}ms`)
      }

      return result
    },
  }
}
```
