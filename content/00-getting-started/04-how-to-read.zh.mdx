## `<V7Note>` — Vite 7 差异折叠块

本站最核心的约定。点击下面的标题栏展开看效果:

<V7Note title="Vite 7:双引擎架构">
  Vite 7 的 dev 模式使用 esbuild(Go)做 transform,build 模式使用 Rollup(JS)打包。两套引擎的 hook 行为有细微差异,某些 Rollup 插件在 dev 下不生效。

**迁移到 Vite 8**:Rolldown 统一了两个阶段,插件只需适配一套 API。

</V7Note>

**规则**:

- Vite 8 与 Vite 7 **行为一致**的地方 → 不出现 `<V7Note>`
- **有差异**的地方 → 差异描述全部放进 `<V7Note>`,正文只讲 Vite 8
- Vite 6 及更早版本 → 本站**不覆盖**

## 代码块约定

### 文件名标注

```ts filename="vite.config.ts"
// 右上角 filename 标签说明这段代码属于哪个文件
import { defineConfig } from "vite"
export default defineConfig({})
```

### 行高亮

```ts filename="vite.config.ts" {3-5}
import { defineConfig } from "vite"
export default defineConfig({
  base: "/app/", // ← 这三行被高亮
  build: {
    outDir: "output", // ← 重点关注
  },
})
```

### Diff 标注

```ts
export default defineConfig({
  build: {
    outDir: "dist", // [!code --]
    outDir: "output", // [!code ++]
  },
})
```

### Focus 标注

把注意力聚焦到关键代码,其余行自动淡出。悬停可恢复查看。

```ts
import { defineConfig } from "vite"
import react from "@vitejs/plugin-react" // [!code focus]

export default defineConfig({
  plugins: [react()],
})
```
