## Reading conventions at a glance

The main line of this guide is **Vite 8**. Vite 7 differences never bleed into the body — they're folded into `<V7Note>` blocks. If you're still on Vite 7 (or migrating), expand them; if you started on Vite 8, you can skip them entirely.

## `<V7Note>` — Vite 7 diff block

The site's most important convention. Click the header below to expand:

<V7Note title="Vite 7: dual-engine architecture">
  Vite 7's dev mode uses esbuild (Go) for transform, build mode uses Rollup (JS) for bundling. Hook behavior diverges subtly between the two engines, and some Rollup plugins don't take effect in dev.

**Migrating to Vite 8**: Rolldown unifies both stages, so plugins target a single API.

</V7Note>

**Rules**:

- Where Vite 8 and Vite 7 **behave the same** → no `<V7Note>`
- Where they **differ** → the entire diff goes inside `<V7Note>`; the body covers Vite 8 only
- Vite 6 and earlier → **not covered** on this site

## Code-block conventions

### Filename label

```ts filename="vite.config.ts"
// The filename tag in the corner tells you which file this snippet belongs to
import { defineConfig } from "vite"
export default defineConfig({})
```

### Line highlight

```ts filename="vite.config.ts" {3-5}
import { defineConfig } from "vite"
export default defineConfig({
  base: "/app/", // ← these three lines are highlighted
  build: {
    outDir: "output", // ← focus area
  },
})
```

### Diff markers

```ts
export default defineConfig({
  build: {
    outDir: "dist", // [!code --]
    outDir: "output", // [!code ++]
  },
})
```

### Focus markers

Pull attention to the key lines — everything else fades. Hover to restore visibility.

```ts
import { defineConfig } from "vite"
import react from "@vitejs/plugin-react" // [!code focus]

export default defineConfig({
  plugins: [react()],
})
```
