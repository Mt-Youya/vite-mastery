## `transformIndexHtml` hook

**触发时机**:Vite 处理 `index.html`(以及其他 HTML 入口)时。

**触发条件**:dev 和 build 都触发。

**调用约定**:sequential

**参数**:

```ts
transformIndexHtml(
  html: string,
  ctx: {
    path: string           // HTML 文件相对路径,如 "/"
    filename: string       // HTML 文件绝对路径
    server?: ViteDevServer // dev 模式下存在
    bundle?: OutputBundle  // build 模式下存在
    chunk?: OutputChunk    // build 模式下存在
  }
): IndexHtmlTransformResult
```

**返回值**可以是:

```ts
type IndexHtmlTransformResult =
  | string // 返回修改后的完整 HTML 字符串
  | IndexHtmlTransformHook // 返回一组标签描述对象
  | { html: string; tags: IndexHtmlTransformHook }
```

### 模式一:字符串替换

最直接,适合简单替换:

```ts
transformIndexHtml(html) {
  // 替换 HTML 里的占位符
  return html
    .replace("<!-- INJECT_TITLE -->", "<title>My App</title>")
    .replace("%APP_VERSION%", "1.0.0")
},
```

### 模式二:标签注入(推荐)

通过返回标签描述对象,Vite 负责在正确位置插入:

```ts
transformIndexHtml(html, ctx) {
  return {
    html,
    tags: [
      // 在 <head> 末尾注入
      {
        tag: "meta",
        attrs: {
          name: "build-time",
          content: new Date().toISOString(),
        },
        injectTo: "head",
      },
      // 在 <body> 末尾注入 script
      {
        tag: "script",
        attrs: { type: "module" },
        children: `window.__COMMIT__ = "${getGitCommit()}"`,
        injectTo: "body",
      },
      // 在 <head> 开头注入(优先级高的 meta)
      {
        tag: "meta",
        attrs: { charset: "UTF-8" },
        injectTo: "head-prepend",
      },
    ],
  }
},
```

`injectTo` 可以是:

- `"head"`:在 `</head>` 前插入
- `"body"`:在 `</body>` 前插入
- `"head-prepend"`:在 `<head>` 后紧接插入
- `"body-prepend"`:在 `<body>` 后紧接插入

### 根据 dev/build 不同处理

```ts
transformIndexHtml(html, ctx) {
  const tags: HtmlTagDescriptor[] = []

  if (ctx.server) {
    // dev 模式:注入开发工具
    tags.push({
      tag: "script",
      attrs: { src: "/__dev-tools.js", type: "module" },
      injectTo: "body",
    })
  } else {
    // build 模式:注入分析脚本
    tags.push({
      tag: "script",
      children: `window.__ANALYTICS_ID__ = "GA-XXXXXXX"`,
      injectTo: "head",
    })
  }

  return { html, tags }
},
```

### 修改 HTML 里的 `<link>` / `<script>` 顺序

`transformIndexHtml` 可以完整地重写 HTML 字符串,包括重排标签顺序:

```ts
transformIndexHtml(html) {
  // 把所有 CSS link 提前到 head 最顶部(改善 CLS)
  const cssLinks = [...html.matchAll(/<link[^>]+rel="stylesheet"[^>]*>/g)]
    .map(m => m[0])

  let result = html
  cssLinks.forEach(link => {
    result = result.replace(link, "")
  })
  result = result.replace("<head>", `<head>\n    ${cssLinks.join("\n    ")}`)

  return result
},
```

## `handleHotUpdate` hook

**触发时机**:dev 模式下,文件系统检测到文件改变时。

**触发条件**:只在 dev 模式触发,build 不触发。

**调用约定**:sequential

**参数**:

```ts
handleHotUpdate(ctx: HmrContext): Array<ModuleNode> | void | Promise<...>

interface HmrContext {
  file: string                    // 变化的文件绝对路径
  timestamp: number               // 变化时间戳
  modules: Array<ModuleNode>      // Vite 认为受影响的模块列表
  read(): Promise<string>         // 读取文件新内容
  server: ViteDevServer           // dev server 实例
}
```

**返回值**:

- `ModuleNode[]`:手动指定哪些模块需要更新(覆盖 Vite 的默认判断)
- `[]`:空数组 = 不触发任何 HMR
- `void` / `undefined`:使用 Vite 默认的 HMR 行为

### 最简单的用法:为自定义文件类型触发 HMR

```ts
import type { Plugin } from "vite"

export function yamlPlugin(): Plugin {
  return {
    name: "vite-plugin-yaml",
    // ...resolveId, load, transform 处理 .yaml 文件...

    handleHotUpdate({ file, modules, server }) {
      if (!file.endsWith(".yaml")) return

      // 获取这个 .yaml 文件对应的虚拟模块
      const virtualModule = server.moduleGraph.getModuleById(`\0virtual:${file}`)

      if (virtualModule) {
        // 让这个虚拟模块失效
        server.moduleGraph.invalidateModule(virtualModule)
        // 返回需要更新的模块列表
        return [virtualModule]
      }
    },
  }
}
```

### 过滤掉不必要的 HMR

```ts
handleHotUpdate({ file, modules }) {
  // 过滤:测试文件改变不触发 HMR
  if (file.includes("__tests__") || file.includes(".spec.")) {
    return []  // 空数组 = 什么都不做
  }

  // 默认行为
},
```

### 自定义 HMR 消息

有时候文件改变不需要模块热更新,而是要向浏览器发送自定义通知:

```ts
handleHotUpdate({ file, server }) {
  if (file.endsWith("locales.json")) {
    // 翻译文件改变:通知浏览器刷新语言包
    server.hot.send({
      type: "custom",
      event: "locales-update",
      data: { locale: "zh-CN" },
    })
    // 返回 [] 表示不需要模块级 HMR
    return []
  }
},
```

浏览器端:

```ts
if (import.meta.hot) {
  import.meta.hot.on("locales-update", async ({ locale }) => {
    // 重新加载语言包(不需要整页刷新)
    const messages = await fetchMessages(locale)
    i18n.setLocale(locale, messages)
  })
}
```

### 当文件改变影响多个模块时

```ts
handleHotUpdate({ file, modules, server }) {
  if (!file.endsWith("theme.css")) return

  // theme.css 改变影响所有使用了主题变量的模块
  const allThemedModules = [...server.moduleGraph.idToModuleMap.values()]
    .filter(m => m.importedModules.has(server.moduleGraph.getModuleByUrl("/src/theme.css")!))

  // 让这些模块全部失效
  allThemedModules.forEach(m => server.moduleGraph.invalidateModule(m))
  return allThemedModules
},
```
