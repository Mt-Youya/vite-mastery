## @vitejs/plugin-vue 的角色

Vue SFC(`.vue` 文件)不是标准 JavaScript,浏览器无法直接执行。`@vitejs/plugin-vue` 在 `transform` hook 里把 `.vue` 文件编译成 JavaScript:

```vue
<!-- Button.vue -->
<template>
  <button :class="classes" :style="{ color }">{{ label }}</button>
</template>

<script setup lang="ts">
import { computed } from "vue"
const props = defineProps<{ label: string; variant?: "primary" | "secondary" }>()
const classes = computed(() => `btn btn-${props.variant ?? "primary"}`)
const color = ref("blue")
</script>

<style scoped>
.btn {
  padding: 8px 16px;
}
</style>
```

编译后(简化):

```ts
// Button.vue 的编译产物
import { computed, ref } from "vue"

// script setup 的内容被提取为 setup() 函数
const __sfc__ = {
  __name: "Button",
  props: { label: String, variant: String },
  setup(props) {
    const classes = computed(() => `btn btn-${props.variant ?? "primary"}`)
    const color = ref("blue")
    return { classes, color }
  },
  render: /* template 编译后的 render 函数 */
}

// scoped styles 注入(hash ID 防冲突)
__sfc__.__scopeId = "data-v-HASH"

export default __sfc__
```

## SFC 的三块分别如何处理

### `<template>` — 编译为 render 函数

```vue
<template>
  <div v-if="show">{{ message }}</div>
</template>
```

编译后:

```ts
import {
  createElementVNode as _createElementVNode,
  openBlock as _openBlock,
  createElementBlock as _createElementBlock,
  createCommentVNode as _createCommentVNode,
} from "vue"

function render({ show, message }) {
  return show ? (_openBlock(), _createElementBlock("div", null, message)) : _createCommentVNode("v-if", true)
}
```

### `<script setup>` — 编译为 setup() 函数

`<script setup>` 是 Vue 3.2+ 的语法糖,编译后:

```vue
<script setup>
import { ref } from "vue"
const count = ref(0)
defineProps<{ title: string }>()
</script>
```

等价于:

```ts
import { ref } from "vue"

export default {
  props: { title: String },
  setup(props) {
    const count = ref(0)
    return { count }
  },
}
```

### `<style scoped>` — Hash + 属性选择器

```vue
<style scoped>
.button {
  color: red;
}
</style>
```

编译后:

```css
/* 每个 SFC 有唯一 hash */
.button[data-v-1a2b3c4d] {
  color: red;
}
```

同时给组件的根元素注入 `data-v-1a2b3c4d` 属性。

## CSS `v-bind` 原理

```vue
<script setup>
const color = ref("blue")
</script>

<style scoped>
.button {
  color: v-bind(color);  /* 动态绑定 color 变量 */
}
```

编译后:

```ts
// 把 v-bind 的 CSS 变量注入到组件根元素
import { useCssVars } from "vue"

useCssVars((ctx) => ({
  "1a2b3c4d-color": ctx.color,
}))
```

```css
.button[data-v-1a2b3c4d] {
  color: var(--1a2b3c4d-color); /* 引用 CSS 变量 */
}
```

## 推荐配置

```ts filename="vite.config.ts"
import { defineConfig } from "vite"
import vue from "@vitejs/plugin-vue"
import { resolve } from "node:path"

export default defineConfig({
  plugins: [
    vue({
      // 自定义块处理(如 <i18n> 块)
      customElement: false, // 不把 SFC 编译为 Custom Element
    }),
  ],
  resolve: {
    alias: {
      "@": resolve("src"),
    },
  },
})
```

## TypeScript 推荐配置

```json filename="tsconfig.json"
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "strict": true,
    "jsx": "preserve",
    "types": ["vite/client"]
  },
  "include": ["src/**/*.ts", "src/**/*.tsx", "src/**/*.vue"]
}
```
