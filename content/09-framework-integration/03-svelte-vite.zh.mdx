## Svelte 的核心思路

Svelte 与 React / Vue 最大的不同:**编译时框架**。

- React / Vue:运行时框架,需要虚拟 DOM diffing
- Svelte:**编译器**,把 `.svelte` 文件编译成原生 DOM 操作代码

```svelte
<!-- Counter.svelte (Svelte 5 Runes) -->
<script>
  let count = $state(0)
  let doubled = $derived(count * 2)
</script>

<button onclick={() => count++}>
  Count: {count}, Doubled: {doubled}
</button>
```

编译后大致等价于:

```js
// 没有虚拟 DOM!直接 DOM 操作
let count = 0

button.addEventListener("click", () => {
  count++
  // 直接更新 DOM 节点
  buttonText.textContent = `Count: ${count}, Doubled: ${count * 2}`
})
```

## @sveltejs/vite-plugin-svelte 的角色

插件在 `transform` hook 里把 `.svelte` 文件交给 Svelte 编译器:

```ts filename="vite.config.ts"
import { defineConfig } from "vite"
import { svelte } from "@sveltejs/vite-plugin-svelte"

export default defineConfig({
  plugins: [svelte()],
})
```

Svelte 编译器处理:

- `<script>`:提取 JS 逻辑,处理 Runes 语法
- `<template>`:编译为高效的 DOM 操作代码
- `<style>`:提取并 scope(基于组件 hash)

## Svelte 5 Runes 与 Vite

Svelte 5 引入了 Runes 系统——更显式的响应式声明:

```svelte
<script>
  // $state:响应式变量(替代原来的 let)
  let count = $state(0)
  let name = $state("World")

  // $derived:派生值(替代原来的 $: 语句)
  let greeting = $derived(`Hello, ${name}!`)
  let doubled = $derived(count * 2)

  // $effect:副作用(替代原来的 $: 语句块)
  $effect(() => {
    console.log(`count changed to ${count}`)
  })

  // $props:组件 Props
  let { title = "Default" } = $props()
</script>

<h1>{greeting}</h1>
<p>Count: {count}, Doubled: {doubled}</p>
<button onclick={() => count++}>+1</button>
```

## Svelte 的构建优势

Svelte 的编译时特性让它的产物体积通常更小:

| 框架   | 运行时大小               | 组件逻辑   |
| ------ | ------------------------ | ---------- |
| React  | ~45KB(react + react-dom) | 额外 JS    |
| Vue    | ~30KB(vue core)          | 额外 JS    |
| Svelte | ~2KB(极小)               | 编译进组件 |

**注意**:当应用变大,Svelte 的优势会减小(每个组件的编译产物都有重复模式)。
