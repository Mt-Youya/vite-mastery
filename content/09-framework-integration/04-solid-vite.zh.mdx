## SolidJS 的核心原理

Solid 的 JSX 看起来像 React,但编译后完全不同:

```tsx
// 看起来像 React...
function Counter() {
  const [count, setCount] = createSignal(0)

  return (
    <div>
      <p>Count: {count()}</p>
      <button onClick={() => setCount((c) => c + 1)}>+1</button>
    </div>
  )
}
```

编译后(简化):

```ts
// 没有 Virtual DOM!没有组件重渲染!
function Counter() {
  const [count, setCount] = createSignal(0)

  // Solid 的 JSX 只运行一次,创建真实 DOM 节点
  const div = document.createElement("div")
  const p = document.createElement("p")
  const button = document.createElement("button")

  // 创建响应式绑定:count 信号改变时只更新 text node
  insert(p, () => `Count: ${count()}`)

  button.textContent = "+1"
  button.addEventListener("click", () => setCount((c) => c + 1))

  return div
}
```

关键区别:**Solid 的组件函数只运行一次**,不像 React 每次状态改变都重新渲染。

```ts filename="vite.config.ts"
import { defineConfig } from "vite"

export default defineConfig({
  plugins: [solid()],
```

## Solid 核心 API

```tsx
import {
  createSignal, // 基础响应式状态
  createEffect, // 响应式副作用
  createMemo, // 计算值
  Show,
  For, // 控制流组件
} from "solid-js"

function App() {
  const [count, setCount] = createSignal(0)
  const doubled = createMemo(() => count() * 2)

  createEffect(() => {
    // 当 count 改变时执行(注意:读取 count() 建立依赖)
    console.log("count:", count())
  })

  return (
    <div>
      {/* Show:条件渲染(比 && 更精确) */}
      <Show when={count() > 0}>
        <p>Count: {count()}</p>
        <p>Doubled: {doubled()}</p>
      </Show>

      {/* For:列表渲染(带 key 的高效更新) */}
      <For each={["a", "b", "c"]}>{(item) => <span>{item}</span>}</For>

      <button onClick={() => setCount((c) => c + 1)}>+1</button>
    </div>
  )
}
```

## Solid vs React 的关键差异

| 维度        | React                | Solid                |
| ----------- | -------------------- | -------------------- |
| 更新粒度    | 组件重渲染           | 信号级别精确更新     |
| 组件函数    | 每次状态变化重新调用 | 只调用一次           |
| 访问状态    | `state`              | `state()` (函数调用) |
| 条件渲染    | `&&` 或三元          | `<Show>` 组件        |
| 列表渲染    | `Array.map()`        | `<For>` 组件         |
| Virtual DOM | ✅                   | ❌ 原生 DOM          |

3. Solid 的 `<Show>` 组件与 React 的 `&&` 条件渲染有什么本质区别?
4. 在 Solid 里,把 `{count()}` 改写成 `{count}` 会发生什么?

```tsx filename="练习"
// 把以下 React 代码改写为 Solid:

function TodoList() {
  const [todos, setTodos] = useState([
    { id: 1, text: "学习 Vite", done: false },
    { id: 2, text: "学习 Solid", done: false },
  ])
  const [input, setInput] = useState("")

  const completedCount = useMemo(() => todos.filter((t) => t.done).length, [todos])

  const addTodo = () => {
    if (!input.trim()) return
    setTodos([...todos, { id: Date.now(), text: input, done: false }])
    setInput("")
  }

  return (
    <div>
      <p>
        {completedCount}/{todos.length} 完成
      </p>
      <ul>
        {todos.map((todo) => (
          <li key={todo.id}>
            <input
              type="checkbox"
              checked={todo.done}
              onChange={() => setTodos(todos.map((t) => (t.id === todo.id ? { ...t, done: !t.done } : t)))}
            />
            {todo.text}
          </li>
        ))}
      </ul>
      <input value={input} onChange={(e) => setInput(e.target.value)} />
      <button onClick={addTodo}>添加</button>
    </div>
  )
}
```
