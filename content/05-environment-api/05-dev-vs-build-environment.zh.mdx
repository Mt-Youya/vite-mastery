## 两种 Environment 状态

同一个 `environments.ssr` 配置,在 dev 模式和 build 模式下实例化的是不同的类:

```text
vite dev:
  ssr 配置 → DevEnvironment 实例
    ├── moduleGraph (per-environment)
    ├── hot (HotChannel)
    └── runner (ModuleRunner)

vite build:
  ssr 配置 → BuildEnvironment 实例
    ├── bundle (输出产物)
    └── (无 hot / runner)
```

## DevEnvironment:开发态的能力

`DevEnvironment` 是 dev server 运行时的 Environment 实例:

```ts
interface DevEnvironment {
  name: string // "client" | "ssr" | 自定义
  config: ResolvedConfig
  moduleGraph: EnvironmentModuleGraph // 独立的模块图谱

  // 开发态独有:
  hot: HotChannel // HMR 消息通道
  // runner: ModuleRunner         // 模块执行器(独立接口,以官方文档为准)

  // 方法:
  // transformRequest(url): 手动触发模块 transform
}
```

### 访问 DevEnvironment

```ts
configureServer(server) {
  const ssrEnv = server.environments.ssr

  // 检查环境是否存在
  if (!ssrEnv) return

  // 发送 HMR 消息到 ssr 环境
  ssrEnv.hot.send({
    type: "custom",
    event: "ssr:cache-clear",
    data: {},
  })
}
```

### 在插件里区分 DevEnvironment 和 BuildEnvironment

```ts
import type { Plugin, DevEnvironment } from "vite"

export function adaptivePlugin(): Plugin {
  return {
    name: "adaptive-plugin",
    buildStart() {
      // this.environment 是当前 Environment 实例
      const env = this.environment

      if (!env) return

      // 区分 dev 和 build
      if ("hot" in env) {
        // dev 模式:DevEnvironment
        const devEnv = env as DevEnvironment
        console.log(`[${devEnv.name}] dev 模式启动`)
      } else {
        // build 模式:BuildEnvironment
        console.log(`[${env.name}] build 模式启动`)
      }
    },
  }
}
```

## BuildEnvironment:构建态的能力

`BuildEnvironment` 在 `vite build` 时使用:

```ts
interface BuildEnvironment {
  name: string
  config: ResolvedConfig

  // 构建态独有:
  // 产物相关的 API,具体以官方文档为准
}
```

BuildEnvironment 没有 `hot` 和 `runner`——这些概念只在 dev server 有意义。

构建时通过插件的 `generateBundle` 等 hook 处理产物,而不是通过 Environment API 的高层方法。

## 为两种状态编写不同的行为

```ts
import type { Plugin } from "vite"

export function dualModePlugin(): Plugin {
  return {
    name: "dual-mode-plugin",

    // 这两个 hook 在 dev 和 build 都触发
    resolveId(id) {
      /* ... */
    },
    load(id) {
      /* ... */
    },

    // 只在 dev 模式下:配置 server
    configureServer(server) {
      // 注册 dev 专属的中间件
      server.middlewares.use("/api/refresh", (req, res) => {
        res.end("ok")
      })
    },

    // 只在 build 模式下:分析产物
    generateBundle(_, bundle) {
      // 分析 build 产物
      const chunks = Object.values(bundle).filter((c) => c.type === "chunk")
      console.log(`构建产出 ${chunks.length} 个 chunk`)
    },
  }
}
```

## Environment 感知的 transform

在 transform 里,根据是否有 hot 通道判断当前状态:

```ts
transform(code, id) {
  const env = this.environment
  const isDevMode = env && "hot" in env

  if (isDevMode) {
    // 开发模式:注入调试辅助代码
    return {
      code: `
// [dev-only] 开发调试代码
if (import.meta.env.DEV) {
  window.__DEBUG_MODULES__ = window.__DEBUG_MODULES__ || new Set()
  window.__DEBUG_MODULES__.add(${JSON.stringify(id)})
}
${code}
`,
      map: null,
    }
  }

  // 生产模式:不注入调试代码
  return null
},
```
