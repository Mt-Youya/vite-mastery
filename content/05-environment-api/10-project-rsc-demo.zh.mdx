## 项目目标

构建一个最简的 RSC-like 架构,展示 Environment API 的核心价值:

```text
浏览器
  └── client Environment
        └── 普通 React 组件(有状态、事件处理)

Node.js(SSR)
  └── ssr Environment
        └── SSR 入口,调用 rsc environment 获取服务端组件产物

"RSC" 环境(rsc)
  └── rsc Environment
        └── 服务端组件(直接访问数据库/文件系统)
        └── 只允许用 react-server exports
```

## 项目结构

```text
examples/env-api-rsc-demo/
├── vite.config.ts     ← 三环境配置
├── server.js          ← HTTP server + ModuleRunner
├── src/
│   ├── main.tsx       ← client 入口
│   ├── App.tsx        ← 普通 React 组件(client)
│   ├── entry-server.tsx  ← ssr 入口
│   └── server-component.tsx  ← RSC-like 服务端组件
```

## 步骤一:配置三个 Environment

```ts filename="vite.config.ts"
import { defineConfig } from "vite"
import react from "@vitejs/plugin-react"

export default defineConfig({
  plugins: [react()],

  environments: {
    // client:浏览器 React 应用
    client: {
      resolve: {
        conditions: ["browser", "import"],
      },
    },

    // ssr:Node.js SSR 入口
    ssr: {
      resolve: {
        conditions: ["node", "import"],
        noExternal: ["react", "react-dom"],
      },
    },

    // rsc:服务端组件环境
    // 注意:实际 RSC 需要 react-server condition
    // 但这里做简化演示,用 node 即可
    rsc: {
      resolve: {
        conditions: ["node", "import"],
      },
    },
  },
})
```

## 步骤二:服务端组件

```tsx filename="src/server-component.tsx"
/**
 * RSC-like 服务端组件。
 * 只在 rsc environment 里运行。
 * 可以直接访问数据库、文件系统等 server-only 资源。
 */
import { readFileSync } from "node:fs"
import { resolve } from "node:path"

interface Post {
  title: string
  content: string
}

// 模拟从数据库读取数据(实际应用里是真实数据库查询)
async function getPosts(): Promise<Post[]> {
  const dataPath = resolve("data/posts.json")
  try {
    return JSON.parse(readFileSync(dataPath, "utf-8")) as Post[]
  } catch {
    return [
      { title: "示例文章 1", content: "这是来自服务端的内容" },
      { title: "示例文章 2", content: "直接从文件系统读取" },
    ]
  }
}

// 服务端组件:返回序列化的 JSON 数据(而不是 HTML)
export async function ServerPostList(): Promise<string> {
  const posts = await getPosts()
  // 在真实 RSC 里这里会返回 RSC flight 格式
  // 简化版直接返回 JSON
  return JSON.stringify(posts)
}
```

## 步骤三:SSR 入口

```tsx filename="src/entry-server.tsx"
import { renderToString } from "react-dom/server"

interface ServerData {
  posts: Array<{ title: string; content: string }>
}

export async function render(url: string, serverData: ServerData): Promise<string> {
  // 注意:这里的 App 是在 ssr environment 里加载的
  const { App } = await import("./App")

  const appHtml = renderToString(<App posts={serverData.posts} />)

  return `<!doctype html>
<html lang="zh">
  <head>
    <meta charset="UTF-8" />
    <title>Env API RSC 演示</title>
  </head>
  <body>
    <div id="root">${appHtml}</div>
    <script>window.__INITIAL_DATA__ = ${JSON.stringify(serverData)}</script>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>`
}
```

## 关键设计原则

### 原则一:每个环境有自己的模块图

```text
client environment:
  ModuleGraph {
    "/src/App.tsx" → ... (browser resolve)
  }

ssr environment:
  ModuleGraph {
    "/src/App.tsx" → ... (node resolve,可能是不同版本)
    "/src/entry-server.tsx" → ...
  }

rsc environment:
  ModuleGraph {
    "/src/server-component.tsx" → ... (node resolve)
    "node:fs" → external
  }
```

### 原则二:敏感代码隔离

`server-component.tsx` 包含 `readFileSync`——这个模块只应该在 `rsc` 环境里存在。客户端永远不能 import 它:

```text
client 浏览器 → 无法 import server-component.tsx
                (rsc environment 的模块图对 client 不可见)
```
