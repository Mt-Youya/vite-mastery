## buildApp 是什么

`buildApp` 是 Vite 7 新增、Vite 8 进一步成熟的插件 hook,专门用于**编排多环境构建**。

在没有 `buildApp` 之前,如果你的项目有三个环境(client / ssr / rsc),需要分别执行:

```bash
vite build                  # 构建 client
vite build --ssr            # 构建 ssr
# rsc 没有标准方式,框架各自实现
```

`buildApp` 让这个编排过程可以用代码描述:

```ts filename="my-framework-plugin.ts"
import type { Plugin } from "vite"

export function myFrameworkPlugin(): Plugin {
  return {
    name: "my-framework",
    async buildApp(app) {
      // 按顺序构建:先 ssr,再 client(client 可能依赖 ssr 的产物)
      await app.build("ssr")
      await app.build("client")
    },
  }
}
```

<Callout type="warning">
  `buildApp` hook 的具体参数和 `app.build()` 的 API 在 RC 阶段仍在确认中。上面的代码是概念示意,以官方文档为准。
</Callout>

## 典型场景:SSR + Client 顺序构建

许多 SSR 框架需要先构建服务端代码,再构建客户端代码。原因:

1. SSR 产物包含路由清单
2. Client 需要读取路由清单来做代码分割

```ts
async buildApp(app) {
  // 第一步:构建 SSR 环境
  await app.build("ssr")
  // SSR 产物现在在 dist/server/ 里

  // 从 SSR 产物读取路由清单
  const manifest = readRouteManifest("dist/server/route-manifest.json")

  // 第二步:带着路由清单构建 Client
  // (需要某种方式把 manifest 传给 client 构建)
  await app.build("client")
}
```

## 典型场景:并行构建独立环境

如果两个环境没有依赖关系,可以并行:

```ts
async buildApp(app) {
  // client 和 worker 互不依赖,并行构建
  await Promise.all([
    app.build("client"),
    app.build("worker"),
  ])
}
```

## 框架视角:Next.js 类似的构建流程

```ts
async buildApp(app) {
  // 1. 构建 RSC 环境(React Server Components)
  await app.build("rsc")

  // 2. 读取 RSC 构建的 server component manifest
  const rscManifest = readManifest("dist/rsc/manifest.json")

  // 3. 把 manifest 传入 ssr 构建
  await app.build("ssr")

  // 4. 最后构建 client(需要完整的 SSR + RSC manifest)
  await app.build("client")
}
```

## 在 buildApp 里共享信息

```ts
async buildApp(app) {
  const sharedData = new Map<string, unknown>()

  // ssr 构建完成后,收集信息
  await app.build("ssr")
  sharedData.set("routeManifest", readManifest())

  // client 构建时,插件可以访问 sharedData
  // (通过闭包或其他机制传递,具体实现取决于框架)
  await app.build("client")
}
```
