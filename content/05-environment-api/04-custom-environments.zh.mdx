## 什么时候需要自定义环境

内置的 `client` 和 `ssr` 两个环境覆盖了大多数传统应用。以下场景需要自定义环境:

| 场景                        | 需要的自定义环境                  |
| --------------------------- | --------------------------------- |
| Cloudflare Workers          | 使用 `workerd` runtime 的独立环境 |
| React Server Components     | 独立的 `rsc` 环境                 |
| Service Worker              | 在浏览器里但有独立作用域          |
| Edge Middleware(如 Next.js) | 独立的 `edge` 环境                |
| 多 Worker 架构              | 多个 worker 各自一个环境          |

## 基本配置结构

```ts filename="vite.config.ts"
import { defineConfig } from "vite"

export default defineConfig({
  environments: {
    // 默认 client 和 ssr 仍然存在
    client: {},
    ssr: {},

    // 新增:Cloudflare Workers 环境
    worker: {
      resolve: {
        // Cloudflare Workers 使用的 export conditions
        conditions: ["workerd", "browser", "import"],
        // Cloudflare Workers 不能 bundle Node.js 内置模块
        noExternal: true,
      },
    },
  },
})
```

## 为 Cloudflare Workers 配置环境

Cloudflare Workers 运行在 `workerd` runtime——不是 Node.js,也不是浏览器,是专属的 V8 沙箱:

```ts filename="vite.config.ts"
import { defineConfig } from "vite"

export default defineConfig({
  environments: {
    worker: {
      resolve: {
        // workerd 优先,fallback 到 browser
        conditions: ["workerd", "worker", "browser", "import"],
        // 不能使用 Node.js 内置模块(fs, path, crypto 等)
        // 但可以使用 Cloudflare 的 node: 兼容层
        noExternal: true,
        external: [
          // Cloudflare 提供的内置模块
          "__STATIC_CONTENT_MANIFEST",
        ],
      },
      build: {
        // 产物格式:ESM(workerd 原生支持)
        rolldownOptions: {
          output: {
            format: "es",
          },
        },
      },
    },
  },
})
```

## 为 React Server Components 配置环境

RSC 有独特的 resolve 需求——需要 `react-server` condition:

```ts filename="vite.config.ts"
import { defineConfig } from "vite"

export default defineConfig({
  environments: {
    client: {},
    ssr: {},
    rsc: {
      // RSC 专用:react-server condition
      resolve: {
        conditions: ["react-server", "node", "import"],
        noExternal: ["react", "react-dom"],
      },
    },
  },
})
```

`react-server` condition 告诉 React 相关的包使用 Server Component 专属的导出:

```json
// react/package.json(简化)
{
  "exports": {
    ".": {
      "react-server": "./react.server.js", // RSC 专属版本
      "import": "./react.js", // 普通 ESM
      "require": "./cjs/react.js"
    }
  }
}
```

## 为自定义环境编写专属插件

有时候某个插件只应该在特定环境运行:

```ts filename="vite.config.ts"
import { defineConfig } from "vite"
import type { Plugin } from "vite"

function workerOnlyPlugin(): Plugin {
  return {
    name: "vite-plugin-worker-only",
    // 通过 applyToEnvironment 限定运行环境(RC 阶段 API,以官方文档为准)
    transform(code, id) {
      // 只在 worker 环境执行
      if (this.environment?.name !== "worker") return null

      // Worker 专属处理:替换 Node.js API
      return {
        code: code.replace("process.env.NODE_ENV", '"production"'),
        map: null,
      }
    },
  }
}

export default defineConfig({
  plugins: [workerOnlyPlugin()],
  environments: {
    worker: {
      // 或者直接在环境里配置专属插件
      plugins: [workerSpecificPlugin()],
    },
  },
})

function workerSpecificPlugin(): Plugin {
  return { name: "worker-specific" }
}
```

## 访问自定义环境

在 `configureServer` 里访问自定义环境:

```ts
configureServer(server) {
  // 访问 worker 环境
  const workerEnv = server.environments.worker
  if (workerEnv) {
    console.log("worker 环境模块图:", workerEnv.moduleGraph)
  }
}
```
