## SSR 的两种模式

Vite 支持两种 SSR 使用方式:

### 模式一:中间件模式(开发推荐)

```ts filename="server.js"
import { createServer as createViteServer } from "vite"
import express from "express"

const vite = await createViteServer({
  server: { middlewareMode: true },
  appType: "custom", // 不自动注入 HTML
})

const app = express()
app.use(vite.middlewares) // Vite 处理静态资源和 HMR

app.use("*", async (req, res) => {
  const url = req.originalUrl

  try {
    // 加载并执行服务端入口
    const { render } = await vite.ssrLoadModule("/src/entry-server.tsx")
    const appHtml = await render(url)

    // 获取 index.html 并注入渲染结果
    const template = await vite.transformIndexHtml(url, readFileSync("index.html", "utf-8"))
    const html = template.replace("<!--ssr-outlet-->", appHtml)

    res.status(200).set("Content-Type", "text/html").end(html)
  } catch (e) {
    vite.ssrFixStacktrace(e)
    res.status(500).end(String(e))
  }
})
```

### 模式二:预构建 SSR(生产推荐)

```bash
# 构建 client 端
vite build

# 构建 SSR 端(--ssr 标志)
vite build --ssr src/entry-server.tsx
```

产物:

```text
dist/
  client/         ← 浏览器端代码
    index.html
    assets/
  server/         ← Node.js 端代码
    entry-server.js
```

## ssrLoadModule vs Environment API

`vite.ssrLoadModule()` 是 Vite 的传统 SSR API,在 Vite 8 里**仍然有效**但不是推荐方式:

```ts
// 传统方式(仍然有效)
const { render } = await vite.ssrLoadModule("/src/entry-server.tsx")

// Vite 8 推荐(Environment API)
// 以官方文档 API 为准
const { render } = await runner.import("/src/entry-server.tsx")
```

两者的主要区别:

- `ssrLoadModule`:每次调用都重新执行,不缓存
- `ModuleRunner.import()`:有缓存,HMR 更新时才重新执行

## SSR 构建配置

```ts filename="vite.config.ts"
import { defineConfig } from "vite"
import react from "@vitejs/plugin-react"

export default defineConfig({
  plugins: [react()],

  build: {
    // SSR 构建时需要的配置
    // (注意:不要把这些放在顶层,用条件判断或分开的配置文件)
  },

  ssr: {
    // SSR 专属配置
    noExternal: ["some-esm-only-package"], // 需要 bundle 进产物的包
    external: ["express"], // 明确 external(Node.js 内置模块自动 external)
    target: "node", // 目标是 Node.js 环境
  },
})
```

## 错误处理与 stack trace

SSR 报错时,stack trace 会指向 transform 后的代码而不是源码。`ssrFixStacktrace` 修复这个问题:

```ts
} catch (e) {
  vite.ssrFixStacktrace(e)  // 把 stack trace 重写为源码行号
  console.error(e)
  res.status(500).end(String(e))
}
```

## 完整请求流程

```text
浏览器请求 GET /page
      │
      ▼
Express 中间件
      │
      ▼
ssrLoadModule("/src/entry-server.tsx")
  → Vite 在 ssr 环境执行模块
  → 返回 { render } 函数
      │
      ▼
render(url)
  → 调用 React renderToString 或 Vue renderToString
  → 返回 HTML 字符串
      │
      ▼
transformIndexHtml(url, template)
  → 把 Vite 处理的 HTML 模板返回(含资源链接)
      │
      ▼
template.replace("<!--ssr-outlet-->", appHtml)
      │
      ▼
res.end(html)
```
