## 为什么每个 Environment 需要独立的 HotChannel

在 Vite 7(单一模块图)时代,只有一个 HMR 通道——WebSocket,连接到浏览器。

当 Environment API 引入多个独立环境后,不同环境需要不同的 HMR 通信方式:

| 环境   | 通信目标                      | 通信方式                    |
| ------ | ----------------------------- | --------------------------- |
| client | 浏览器                        | WebSocket                   |
| ssr    | Node.js 进程里的 ModuleRunner | 进程内函数调用/EventEmitter |
| worker | workerd 进程                  | HTTP / IPC                  |

`HotChannel` 抽象统一了这些不同的通信方式。

## HotChannel API(概念)

每个 `DevEnvironment` 有一个 `hot` 属性:

```ts
interface HotChannel {
  // 向该环境的客户端发送消息
  send(payload: HotPayload): void

  // 订阅来自客户端的消息
  on(event: string, listener: (data: unknown) => void): void
  off(event: string, listener: (data: unknown) => void): void
}
```

<Callout type="warning">以上是概念性描述。具体接口以 Vite 8.1.x 官方文档为准。</Callout>

## client 环境的 HotChannel

`client` 环境的 HotChannel 是基于 WebSocket 实现的:

```ts
configureServer(server) {
  // 向所有浏览器 client 发送消息
  server.environments.client.hot.send({
    type: "custom",
    event: "my-event",
    data: { key: "value" },
  })

  // 订阅来自浏览器的消息(较少见)
  server.environments.client.hot.on("browser-message", (data) => {
    console.log("Browser sent:", data)
  })
},
```

这等价于直接用 `server.hot.send()`——`server.hot` 默认就是 client 环境的 HotChannel。

## ssr 环境的 HotChannel

`ssr` 环境的 HotChannel 不通过 WebSocket——因为 ssr 代码运行在 Node.js 里,同进程通信更高效:

```ts
configureServer(server) {
  // 通知 ssr 环境的 ModuleRunner 某些模块需要失效
  server.environments.ssr?.hot.send({
    type: "custom",
    event: "ssr:invalidate",
    data: { path: "/src/data-fetcher.ts" },
  })
},
```

ModuleRunner 监听这个消息并使对应模块失效:

```ts
// 在 ModuleRunner 初始化时配置 HotChannel 监听

```

## 在插件里跨环境发送 HMR 消息

```ts
import type { Plugin } from "vite"

export function crossEnvHmrPlugin(): Plugin {
  return {
    name: "cross-env-hmr",
    handleHotUpdate({ file, server }) {
      if (!file.endsWith(".data.ts")) return

      // 同时通知 client 和 ssr 环境
      server.environments.client.hot.send({
        type: "custom",
        event: "data-update",
        data: { file },
      })

      server.environments.ssr?.hot.send({
        type: "custom",
        event: "ssr:data-update",
        data: { file },
      })

      return []
    },
  }
}
```

## 自定义环境的 HotChannel

对于 Cloudflare Workers 等自定义环境,需要实现自定义的 HotChannel。这通常由框架插件(如 `@cloudflare/vite-plugin`)来做,不需要应用开发者手动处理。

```ts
// 概念:自定义 HotChannel(框架开发者实现)
class WorkerdHotChannel implements HotChannel {
  private connection: WorkerdConnection

  send(payload: HotPayload) {
    // 通过 HTTP 或 IPC 把消息发给 workerd 进程
    this.connection.postMessage(JSON.stringify(payload))
  }

  on(event: string, listener: (data: unknown) => void) {
    this.connection.on(`message:${event}`, listener)
  }

  off(event: string, listener: (data: unknown) => void) {
    this.connection.off(`message:${event}`, listener)
  }
}
```

const THEME_FILE = resolve("theme.json")
