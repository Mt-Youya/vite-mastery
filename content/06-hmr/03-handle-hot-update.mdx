## `handleHotUpdate` 参数详解

```ts
handleHotUpdate(ctx: HmrContext): Array<ModuleNode> | void | Promise<...>

interface HmrContext {
  file: string                    // 变化的文件绝对路径
  timestamp: number               // 变化时间戳(毫秒)
  modules: Array<ModuleNode>      // Vite 推断的受影响模块列表
  read(): Promise<string>         // 异步读取文件新内容
  server: ViteDevServer           // dev server 实例
}
```

**返回值**:

- `ModuleNode[]` — 覆盖 Vite 的默认判断,只更新这些模块
- `[]` — 空数组,不触发任何 HMR(但也不整页刷新)
- `void` — 使用 Vite 的默认行为

## 案例一:为 YAML 文件实现 HMR

假设你有一个 YAML 加载插件,让 `.yaml` 文件作为 ES Module 使用:

```ts filename="yaml-plugin.ts"
import type { Plugin } from "vite"
import { readFileSync } from "node:fs"

function parseYaml(content: string): unknown {
  // 假设你已经有 yaml 解析器
  return {} // 占位
}

export function yamlPlugin(): Plugin {
  return {
    name: "vite-plugin-yaml",

    load(id) {
      if (!id.endsWith(".yaml")) return null
      const content = readFileSync(id, "utf-8")
      const data = parseYaml(content)
      return `export default ${JSON.stringify(data)}`
    },

    handleHotUpdate({ file, modules, server }) {
      if (!file.endsWith(".yaml")) return

      // Vite 默认会把 .yaml 文件的变化传给 modules
      // 直接返回 modules 即可触发这些模块的 HMR
      console.log(`[yaml-plugin] 热更新: ${file}`)
      return modules
    },
  }
}
```

## 案例二:配置文件变化,发送自定义事件

当配置文件改变时,不做模块级 HMR,而是向浏览器发送自定义事件:

```ts filename="config-plugin.ts"
import type { Plugin } from "vite"
import { readFileSync } from "node:fs"
import { resolve } from "node:path"

export function configPlugin(): Plugin {
  const configPath = resolve("app.config.json")

  return {
    name: "vite-plugin-config",

    load(id) {
      if (id !== "\0virtual:config") return null
      this.addWatchFile(configPath)
      return `export default ${readFileSync(configPath, "utf-8")}`
    },

    resolveId(id) {
      if (id === "virtual:config") return "\0virtual:config"
      return null
    },

    handleHotUpdate({ file, server }) {
      if (file !== configPath) return

      // 配置变化:通知浏览器,让它自己决定如何响应
      server.hot.send({
        type: "custom",
        event: "config-update",
        data: {
          timestamp: Date.now(),
          file: file.replace(process.cwd(), ""),
        },
      })

      // 同时让虚拟配置模块失效
      const module = server.moduleGraph.getModuleById("\0virtual:config")
      if (module) {
        server.moduleGraph.invalidateModule(module)
        return [module]
      }

      return []
    },
  }
}
```

浏览器端:

```ts
// src/main.ts
import config from "virtual:config"

if (import.meta.hot) {
  import.meta.hot.on("config-update", async () => {
    // 重新动态 import 最新配置
    const newConfig = await import("virtual:config")
    applyConfig(newConfig.default)
    console.log("[HMR] 配置已热更新")
  })
}
```

## 案例三:批量文件变化的防抖

某些场景下(比如 `pnpm install` 更新了大量文件),HMR 可能被触发几百次。防抖处理:

```ts
import type { Plugin } from "vite"

export function debouncedHmrPlugin(): Plugin {
  let timer: ReturnType<typeof setTimeout> | null = null
  let pendingModules: Set<string> = new Set()

  return {
    name: "debounced-hmr",
    handleHotUpdate({ file, modules, server }) {
      pendingModules.add(file)

      if (timer) clearTimeout(timer)

      timer = setTimeout(() => {
        // 统一处理所有待更新的文件
        console.log(`[hmr] 批量更新 ${pendingModules.size} 个文件`)
        pendingModules.clear()
        timer = null
        // 触发页面刷新
        server.hot.send({ type: "full-reload" })
      }, 300)

      return [] // 阻止立即 HMR,等待防抖完成
    },
  }
}
```

## 案例四:测试文件不触发 HMR

```ts
handleHotUpdate({ file, modules }) {
  // 测试文件改变,不触发任何更新
  if (
    file.includes("__tests__") ||
    file.includes(".spec.") ||
    file.includes(".test.")
  ) {
    console.log(`[hmr] 跳过测试文件: ${file.split("/").pop()}`)
    return []
  }
},
```
